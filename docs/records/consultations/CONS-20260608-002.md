### 4 ROOT CAUSES

**Root Cause 1 — Drug OOV Hallucination:**
- PhoWhisper trained trên general Vietnamese (CommonVoice VN, FLEURS)
- Drug names = English/Latin OOV → decoder maps to nearest VN phoneme sequence
- Kết quả: deterministic hallucination ("Ciprofloxacin" → always "karaoke-like" output)
- L1b fuzzy matching chỉ works khi phonetic distance < threshold → hoàn toàn fail với hallucination

**Root Cause 2 — Synthetic Training Distribution Mismatch:**
- PhoBERT trained trên perfect, well-formed sentences
- Real: run-on sentences + disfluency + abbreviations + fast speech
- "glibizi glibizi" ≠ "Glipizide" — L1b needs to see this pattern
- "hát 81mg" ≠ "Aspirin 81mg" — 2/5 characters match, below threshold

**Root Cause 3 — No Clinical Language Model in ASR Decoder:**
- Whisper decoder uses general Vietnamese language model probability
- Medical context ("kê thuốc", "uống mg") should shift probability toward drug vocabulary
- Currently zero drug-domain LM signal during decoding
- Simple fix available: initial_prompt injection (không cần training)

```
Audio → PhoWhisper (general VN) → raw transcript
      → L1b FuzzyMatch (post-correction, text-level)
      → LangChain/LLaMA NER (general LLM + drug/ICD RAG context)
      → Form fill
```

### Vấn đề:
- ASR error xảy ra ở bước 1 → không thể recover ở bước 2 nếu hallucination
- L1b RAG = text-level knowledge grounding (tốt khi ASR output có overlap)
- LLM NER = general model, không có medical VN specialization
- Không có feedback loop từ BS corrections về ASR

## CÁC PHƯƠNG ÁN KỸ THUẬT CẦN ĐÁNH GIÁ

### Option A: Whisper Prompt Engineering (Zero cost, immediate)
```python
# Inject drug vocabulary vào initial_prompt
initial_prompt = "Bác sĩ kê đơn thuốc: Metformin, Amlodipine, Glipizide, 
                  Ciprofloxacin, Paracetamol, Amoxicillin, Aspirin, 
                  Losartan, Atorvastatin, Prednisolone..."
# Whisper autoregressive decoder sẽ bias toward these tokens
```
- Effort: 2 giờ | Expected WER improvement: 10-20% cho drugs | Risk: thấp
- Tested in literature: ~15% relative WER reduction cho domain-specific vocabulary

### Option B: LoRA Fine-tune PhoWhisper (Medium effort)
```
Data needed: 50-100h medical Vietnamese audio with transcripts
Hardware: RTX 3060 (6GB VRAM) hoặc VNG Cloud A10G
LoRA rank: r=32, alpha=64 → ~3M trainable params (0.4% of 769M)
Training time: ~8-12h on A10G
Expected WER: 35-50% → 15-25% (medical domain)
```
- Effort: 2-3 tuần (data prep + train) | Risk: medium (cần pilot audio)

### Option C: Replace ASR — Conformer-based (High effort)
```
Models to evaluate:
- Wav2Vec2-BERT Vietnamese (HF: nguyenvulebinh/wav2vec2-base-vietnamese-250h)
- Conformer-CTC (NVIDIA NeMo, trained on FLEURS-VN)  
- MMS (Meta, 1100 languages, VN support)
- Whisper-large-v3 vs PhoWhisper-medium comparison
```
- Effort: 1 tuần evaluation | Risk: may not improve without domain fine-tune

### Option D: Two-Stage ASR with Medical LM Reranking
```
Stage 1: PhoWhisper → N-best hypotheses (beam_size=10)
Stage 2: Medical VN language model reranks hypotheses
         → hypothesis với drug names có LM score cao hơn
Medical LM: fine-tune GPT-2 VN trên medical text corpus
```
- Effort: 3-4 tuần | Expected improvement: 20-30%

### Option E: Phoneme-Level Drug Matching (Medium effort)
```
Problem: "karaoke tamon" ↔ "Ciprofloxacin" — text similarity low
Solution: Convert both to phoneme sequences → phoneme edit distance
VN phoneme of "karaoke tamon" → /ka ra o ke ta mon/
Drug phoneme DB: "Ciprofloxacin" → /si prəʊ flɒk sæ sɪn/
→ still different, but better than character-level fuzzy
Better approach: train acoustic embedding → drug embedding similarity
```
- Effort: 1 tuần (G2P + phoneme matching) | Improvement: unknown without testing

### Option F: Replace NER — Fine-tuned Medical VN LLM
```
Options:
- Qwen2.5-7B-Instruct LoRA trên clinical VN data → local inference
- Vistral-7B (VN LLM, LoRA capable) trên medical corpus
- vinai/PhoGPT-7.5B (Vietnamese GPT) + medical fine-tune
Requirement: structured JSON output + hallucination resistance
```
- Effort: 2-4 tuần | Risk: high (need curated medical VN instruction data)

### Option G: vLLM + TensorRT for Production (Low priority now)
```
Current: Groq free tier (rate limited, remote, no customization)
Future (>100 users): 
- vLLM local serving: Qwen2.5-7B-Instruct on RTX 4090 → ~50 token/s
- TensorRT-LLM: 2-4x faster than vLLM on same hardware
- ONNX export PhoWhisper → OpenVINO for CPU → 2x faster inference
```
- Priority: Phase 2 (after pilot GO decision)
